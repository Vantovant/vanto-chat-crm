## 1. Purpose

Document repository that provides the grounding context layer for the Zazi Copilot AI. Supports text ingestion with client-side chunking, full-text search via PostgreSQL `tsvector`, and collection-based organization with strict/assisted modes controlling AI behavior.

---

## 2. Architecture

| Layer | Detail |
|-------|--------|
| Component | `KnowledgeVaultModule` |
| Tables | `knowledge_files`, `knowledge_chunks` |
| Search | `search_knowledge` PostgreSQL function (full-text search) |
| Edge Function | `knowledge-search` |
| Storage | `knowledge-vault` bucket (private, not currently used for text ingestion) |
| Chunking | Client-side JavaScript, ~2000 chars per chunk with 200-char overlap |

## 3. Collections

| ID | Label | Mode | AI Behavior |
|----|-------|------|-------------|
| `general` | General Knowledge & App Manual | Assisted | AI can paraphrase and expand |
| `opportunity` | Business Opportunity | Strict | AI must quote verbatim only |
| `compensation` | Compensation | Strict | No paraphrasing allowed |
| `products` | Product Prices & Benefits | Strict | Exact pricing required |
| `orders` | Orders & Deliveries | Strict | Factual accuracy critical |
| `motivation` | MLM & Wellness Motivation | Assisted | AI can elaborate freely |

### Mode Semantics
- **Strict (🛡️):** Zazi Copilot must only quote directly from chunks. No hallucination or paraphrasing allowed. Used for factual content where accuracy is critical (prices, compensation, orders).
- **Assisted:** Zazi can paraphrase, elaborate, and combine information from chunks. Used for general knowledge and motivational content.

| Expiry Date | Date input | No | When document expires |

### 4.3 Processing Pipeline
```
1. Read raw text (paste or file.text())
2. JSON files: pretty-print via JSON.parse → JSON.stringify
3. cleanText():
   - Remove null bytes (\0)
   - Remove control characters (\x01-\x08, \x0B, \x0C, \x0E-\x1F)
   - Normalize line endings (CRLF → LF)
   - Collapse multiple spaces/tabs
   - Collapse 3+ newlines to 2
   - Trim
4. Validate: minimum 10 characters
5. Create knowledge_files record (status: 'processing')
6. chunkText(): split into ~2000-char chunks with 200-char overlap
7. Insert chunks to knowledge_chunks in batches of 50
8. Update file status: 'approved' (success) or 'rejected' (failure)
```

## 5. Chunking Algorithm

### 5.1 Parameters
| Parameter | Default | Purpose |
|-----------|---------|---------|
| `maxChars` | 2000 | Maximum characters per chunk |
| `overlap` | 200 | Character overlap between consecutive chunks |

### 5.2 Algorithm
```javascript
function chunkText(text, maxChars=2000, overlap=200):
  chunks = []
  start = 0
  while start < text.length:
    end = min(start + maxChars, text.length)
    
    // Try to break at sentence boundary
    if end < text.length:
      lastPeriod = text.lastIndexOf('.', end)
      if lastPeriod > start + maxChars/2:
        end = lastPeriod + 1
    
    chunks.push(text[start:end].trim())
    start = end - overlap
    if start >= text.length: break
  
  return chunks.filter(c => c.length > 10)
```

### 5.3 Chunk Storage
Each chunk record contains:
| Field | Value |
|-------|-------|
| `file_id` | Parent file UUID |
| `chunk_index` | Sequential order (0, 1, 2...) |
| `chunk_text` | The text content |
| `token_count` | Estimated: `ceil(chunk.length / 4)` |
| `search_vector` | Auto-generated by `knowledge_chunks_search_trigger` |

## 7. Search Tab

### 7.1 Search Interface
- Full-text search input with Enter-to-search
- Collection filter applied (if not "All")
- Search button with loading state

### 7.2 Search Flow
1. User enters query
2. Calls `knowledge-search` edge function with:
   ```json
   { "query": "search text", "collection": "products|null", "max_results": 10 }
   ```
3. Edge function calls `search_knowledge` PostgreSQL function
4. Results displayed as cards with:
   - File title + collection icon
   - Collection badge
   - Relevance score
   - Chunk text preview (first 500 chars)

### 7.3 PostgreSQL Search Function
```sql
search_knowledge(query_text, collection_filter, max_results):
  - Joins knowledge_chunks with knowledge_files
  - Filters: status='approved', not expired, optional collection
  - Uses ts_rank with plainto_tsquery for relevance scoring
  - Returns: chunk_id, file_id, chunk_text, file_title, file_collection, relevance
```
